📝 2. 효과적인 프롬프트 작성법

✅좋은 예

  • 원하는 작업의 맥락을 설명
  • 구체적으로 지시
  • 오류 메시지 포함: AI가 문제를 정확히 파악
  • 예상되는 결과나 형태 명시
"C++ 로그 파싱 클래스에서 메모리 누수가 발생합니다. valgrind 결과: definitely lost: 1,024 bytes LogParser.cpp 파일을 확인해주세요."

❌나쁜 예

  • "알아서 해줘"처럼 너무 추상적
  • 맥락 없이 코드만 붙여넣기
  • 원하는 결과를 명시하지 않음
"코드가 안 돼요" "이거 좀 고쳐주세요" "에러 나요"

📂 6. 여러 파일 컨텍스트 제공하기

📝 제공 방법

  1. 여러 파일 열기: LogParser.hpp, main.cpp 등 관련 파일들을 동시에 오픈
  2. @파일명 입력: Cursor Chat에서 @기호와 함께 파일명 입력
  3. 드래그 앤 드롭: 파일을 채팅창으로 직접 끌어다 놓기
  4. 폴더 전체 선택: 프로젝트 디렉토리 전체를 컨텍스트로 제공

🎯 효과

  • 🔗파일 간 관계 이해: 클래스 상속, 함수 호출 관계 파악
  • 🎯정확한 답변: 전체적인 코드 구조를 고려한 조언
  • ⚡효율적 디버깅: 연관된 파일까지 함께 분석

🚨 8. 프로젝트 오픈 시 필수 체크

⚠️ 핵심 체크포인트

Cursor 설정 → 코드베이스 인덱싱 여부 확인

🔍 확인 방법

  1. Cursor 우하단의 인덱싱 상태 확인
  2. "Indexing..." 표시가 완료될 때까지 대기
  3. 모든 프로젝트 파일이 AI에게 인식되었는지 확인
  4. 큰 프로젝트의 경우 수분의 시간이 소요될 수 있음

📈 인덱싱 완료 시 혜택

  • 프로젝트 전체 구조 이해
  • 파일 간 의존성 파악
  • 더 정확한 코드 제안

🛠 10. 문제 해결 시 프롬프트 핵심 요소

🔧 필수 포함 요소

1️⃣ 발생한 오류 메시지

error: use of undeclared identifier 'LogEntry' fatal error: 'LogParser.hpp' file not found Segmentation fault (core dumped)

2️⃣ 문제 발생 상황 설명

  • 전후 코드 흐름
  • 빌드 시 상황
  • 실행 환경 정보
  • 재현 단계

✅ 효과적인 문제 해결 프롬프트 예시

"CMake 빌드 중 다음 오류가 발생했습니다: [오류 메시지] fatal error: 'LogParser.hpp' file not found [상황] - Ubuntu 20.04, GCC 9.4.0 환경 - CMakeLists.txt에서 include_directories() 설정 완료 - 파일은 src/ 디렉토리에 존재함 - 이전 빌드는 성공했으나 파일 이동 후 오류 발생 [시도한 해결책] 1. make clean 후 재빌드 시도 2. 파일 경로 재확인 완료 해결 방법을 제안해주세요."
