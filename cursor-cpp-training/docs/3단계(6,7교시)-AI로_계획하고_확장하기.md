# 🤖 3단계: AI 협업으로 문서 기반 개발 자동화

> **학습 목표**: 2단계에서 직접 작성했던 '작업 지시서'마저 **AI를 활용해 자동 생성**하는 방법을 배웁니다. 이를 통해 '분석 → 계획(문서) 수립 → 검토 → 실행'이라는 전문적인 개발 워크플로우 전체를 AI와 함께 수행합니다.

## Workflow: AI 기반 문서 중심 개발

1.  **(분석)** AI에게 현재 코드나 요구사항 분석을 요청한다.
2.  **(계획)** 분석 결과를 바탕으로, AI에게 상세한 **작업 지시서(MD 문서)** 생성을 요청한다.
3.  **(검토)** AI가 만든 작업 지시서를 사람이 검토하고 수정하며 최종 계획을 확정한다.
4.  **(실행)** 확정된 작업 지시서 문서를 AI에게 전달하여 코드를 생성/수정한다.

---

## 📋 실습 1: AI로 작업 지시서 자동 생성

### 실습 시나리오

"날짜별 로그 그룹핑"이라는 새로운 기능을 추가하려고 합니다. 이 작업을 위한 작업 지시서를 AI에게 만들게 해봅시다.

### 프롬프트 예시:

```
🎯 따라해보세요:

"현재 C++ 프로젝트에 '날짜별 로그 그룹핑' 기능을 추가하고 싶어.
이 작업을 위한 상세한 '작업 지시서'를 마크다운으로 작성해줘.

문서에는 다음 내용이 포함되어야 해:
- 목표
- 기술적인 접근 방법 제안
- 수정해야 할 클래스와 함수 목록
- 예상되는 코드 변경점 요약
- 필요한 테스트 케이스 제안"
```

**기대 결과**: 사람이 직접 고민해야 했던 상세 계획을 AI가 체계적인 문서로 만들어줍니다. 우리는 이 문서를 검토하고, 수정할 부분이 있다면 AI와 대화하며 완성도를 높인 후, 최종 문서를 이용해 코드 생성을 요청하면 됩니다.

## 📊 실습 2: 코드 분석 기반 아키텍처 문서 생성

### 실습 시나리오

기존 C++17 리팩터링 코드를 분석해서 아키텍처 문서를 자동 생성합니다.

### 프롬프트 예시:

```
🎯 따라해보세요:

"현재 프로젝트의 모든 C++ 코드를 분석해서 다음 형식의 아키텍처 문서를 생성해주세요:

1. 시스템 개요 (목적, 주요 기능)
2. 클래스 다이어그램 (관계도 포함)
3. 데이터 흐름 설명
4. 주요 디자인 패턴
5. 확장 포인트 식별
6. 성능 고려사항

각 섹션은 마크다운 형식으로 작성하고,
코드 예시와 다이어그램을 포함해주세요."
```

### 기대 결과:

- 전문적인 수준의 아키텍처 문서
- PlantUML 다이어그램 자동 생성
- 코드 인용과 설명이 조화된 문서

## 📋 실습 3: 테스트 케이스 자동 추출 및 문서화

### 프롬프트 예시:

```
🎯 따라해보세요:

"기존 Catch2 테스트 코드를 분석해서 다음을 포함한
종합적인 테스트 문서를 생성해주세요:

1. 테스트 커버리지 분석
2. 각 클래스별 테스트 시나리오 정리
3. 경계값 테스트 케이스 식별
4. 부족한 테스트 케이스 제안
5. 테스트 실행 가이드
6. CI/CD 연동 방안

실제 코드를 인용하면서 설명해주세요."
```

### 고급 프롬프트:

```
🎯 더 도전해보세요:

"코드 커버리지를 높이기 위한 추가 테스트 케이스를
자동으로 생성하고, 각 테스트의 목적과 중요도를
설명하는 문서를 만들어주세요.

특히 다음 영역에 집중해주세요:
- 에러 처리 경로
- 경계값 조건
- 동시성 이슈
- 메모리 관리"
```

## 🚀 최종 통합 미션: AI 풀스택 개발 (C++ 백엔드 + 웹 UI)

지금까지 우리는 C++ 백엔드 코드를 만들었습니다. 이제 이 데이터를 시각화하는 웹 프론트엔드까지 AI에게 만들어달라고 요청하여 완전한 애플리케이션을 완성해봅시다.

### 실습 시나리오

C++ 프로그램이 생성한 `results.json` 파일을 읽어서, 동적인 웹 대시보드를 생성합니다.

### 프롬프트 예시:

```
🎯 최종 미션 프롬프트:

"프로젝트 루트에 `dashboard.html` 파일을 새로 만들어줘.
이 HTML 파일은 `results.json` 파일을 읽어서 로그 분석 결과를 시각화해야 해.

요구사항:
1.  **디자인**: 전문가 수준의 깔끔하고 모던한 UI로 만들어줘.
2.  **통계 차트**: 로그 레벨별 통계를 보여주는 막대그래프를 그려줘. (Chart.js 라이브러리 사용)
3.  **전체 로그 뷰어**: 모든 로그 목록을 보여주는 테이블이나 리스트를 만들어줘.
4.  **동적 필터링**:
    - 로그 레벨(ERROR, INFO 등)로 필터링하는 드롭다운 메뉴
    - 키워드로 로그 메시지를 검색하는 검색창
5.  모든 기능은 이 HTML 파일 하나에서 동작해야 해. (CSS, JS 포함)"
```

**기대 결과**: C++ 지식만 있던 개발자가 AI의 도움을 받아, 단 한 번의 요청으로 완벽한 프론트엔드 결과물까지 얻어내는 'AI 풀스택 개발'을 경험하게 됩니다.
