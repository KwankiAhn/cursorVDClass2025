# 📝 2단계: MD 작업 지시서 직접 작성하기

> **학습 목표**: 즉흥적인 질문을 넘어, **체계적인 작업 계획서(MD 문서)**를 직접 작성하고 이를 기반으로 AI에게 코드를 요청하는 방법을 익힙니다. 생각을 먼저 정리한 후 AI에게 전달하면 훨씬 정확한 결과물을 얻을 수 있습니다.

## 📋 실습 0: 간단한 작업 지시서 작성 및 활용

리팩터링 실습을 위해, "JSON 출력 기능"을 추가하는 간단한 작업 지시서를 직접 만들어 봅시다.

### 1. (수동) 작업 지시서 작성

아래처럼 간단한 마크다운 파일을 하나 만듭니다. (예: `docs/mission-json.md`)

```markdown
# 작업 지시서: JSON 출력 기능 추가

- **목표**: 로그 분석 결과를 JSON 형식으로 파일에 저장하는 기능을 추가한다.
- **세부 요구사항**:
  1. `--output-json <파일경로>` 라는 새로운 커맨드라인 인자를 받는다.
  2. 이 옵션이 주어지면, `LogStats`의 통계 정보를 JSON 형식으로 변환한다.
  3. 변환된 JSON 데이터를 지정된 파일 경로에 저장한다.
  4. 모든 로그 데이터도 JSON에 포함시킨다.
- **수정 대상 파일**:
  - `main.cpp`: 커맨드라인 인자 처리
  - `LogStats.hpp`: `Statistics` 구조체에 로그 목록 추가
  - `LogStats.cpp`: JSON 변환 로직 수정 및 전체 로그 포함
```

### 2. (AI 활용) 작업 지시서 기반 코드 생성

이제 이 작업 지시서 파일을 AI에게 주면서 코드를 요청합니다.

```
📝 따라해보세요:

"@/docs/mission-json.md
이 작업 지시서의 내용대로 코드를 수정해줘."
```

**기대 결과**: AI가 작업 지시서의 요구사항을 정확히 이해하고, 관련된 여러 파일(`main.cpp`, `LogStats.hpp`, `LogStats.cpp`)을 동시에 수정해주는 것을 확인할 수 있습니다. 이렇게 문서를 통해 명확한 컨텍스트를 제공하면, 훨씬 복잡한 작업도 한번에 처리할 수 있습니다.
